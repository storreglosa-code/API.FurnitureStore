

API MASTERY ROADMAP: 


-------------REGISTRATION-------------

Input: Email y Password
Validar input:
-No es valido --> Bad Request
-Es valido --> Validar si Email existe

Validar Email:
-Si ya existe: Bad Request-> Email already exists
-No existe: Crear usuario, guardar nombre, email y password

Validar Password (nros, mayusc, cant. caracteres):
-No es valida: Bad Request->Invalid Password
-Es valida: Crear Access Token-->Acceso concedido (!!!) 
Despues de tener JWT cambia! -> Si es valido se envia un email de confirmacion.


-------------LOGIN-------------

Input: Email y Password
Validar input:
-No es valido --> Bad Request
-Es valido --> Validar si usuario existe

Usuario existe?
-No existe: Invalid Payload
-Existe: Validar email y password

Mail Confirmado?
-No: necesita ser confirmado
-Si: Validar Email y password.

Email y Pass Validos?
-No son validos: Invalid Credentials.
-Son validos: Crear Access Token --> Acceso concedido
